# WG Blend Forecast Source Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Recreate Windguru's WG blend server-side as the default forecast source, degrading gracefully (blend → AROME-FR → GFS → micro-GFS), with a model param on the API and a forecast-source picker on the watch.

**Architecture:** Spec 2026-07-10-wg-blend-design.md; probe evidence in docs/windguru-api-notes.md ("WG model findings"). New server units: `BlendConfig` (live koef weights via `q=forecast_spot`, snapshot fallback), `Blend` (pure math: hourly grid, within-horizon interpolation, weighted mean, vector-mean direction), constituent fetch/caching + ladder woven into the existing adapter/Conditions path keyed on the verbatim requested-model descriptor. API: `model` param + `requested_model` + `available_models`. Watch: optional decodes, `model` threading (the `stationID` pattern), Settings picker, served≠requested caption.

**Tech Stack:** existing — Elixir (server/), Swift 6 (watch/). No new dependencies either side.

## Global Constraints

- Model param values: `wg` (default when absent; unknown values → treated as `wg`) | `52` | `104` | `117` | `64` | `3`.
- Ladder, each transition logged once via `Logger.warning`: WG blend (≥ 2 constituents) → AROME-FR `52` → GFS `3` → existing micro fallback. Served source ALWAYS reported via the existing opaque `forecast.model` field: `"WG blend (N models)"` / constituent `model_name` strings.
- Blend math: common hourly grid now → +48 h; per-model linear interpolation WITHIN its own horizon only (no extrapolation); wind/gust = koef-weighted arithmetic mean; direction = koef-weighted vector mean (`atan2` of weighted unit-vector sums, normalized to [0, 360); zero-magnitude → highest-weight contributing model's direction at that step); grid steps with < 2 contributing models dropped.
- Weights: fetched from `q=forecast_spot&id_spot=$WG_SPOT_ID` (env, default `1189718`) at boot + every 24 h; wind constituents only (exclude the tab's `id_model_wave`); hardcoded snapshot fallback (captured in Task 1). Weights normalized over models that delivered.
- Per-location constituent availability: "outside grid"/error → cached unavailable for **6 h** keyed by 0.1°-rounded (lat, lon) cell + model id.
- Constituent forecasts cached under the existing 15-min forecast TTL, keyed (position, model id); fetches spaced 2–3 s; one model's failure never refetches others.
- Forecast cache keyed on the VERBATIM requested-model descriptor (repeated identical requests cache-stable; any change invalidates immediately — the station-override lesson).
- All new payload/model fields optional/backward-compatible; new Swift init params appended LAST with `= nil` defaults.
- Server norms: `mix test --warnings-as-errors` + `mix format --check-formatted`; watch norms: `swift test` zero warnings, `xcodebuild ... generic/platform=watchOS Simulator` builds. **No new app-target files → NO xcodegen regeneration → scheme untouched.**
- Secrets discipline for live captures: cookie via `server/.env.capture`, never printed/committed; fixtures sanitized (grep against env values).
